Module Storage (Core Provider)

Trình quản lý lưu trữ tập trung, đóng gói tương tác với Supabase Storage thông qua interface IStorageProvider.

Thông tin Module

Module Name: StorageModule

Files: src/storage/storage.service.ts
src/storage/storage-cleanup.service.ts

Interface: IStorageProvider

Dependencies & Impact

External: @supabase/supabase-js, fs

Impact: Cung cấp khả năng lưu trữ file MP3 cho DownloaderModule và xóa file cho AdminModule.

Sơ đồ Hệ Thống (System Data Flow)

Quy trình khởi tạo

graph TD
A[Load ENV] --> B{URL & Key Valid?}
B -- No --> C[Log Error]
B -- Yes --> D[Connect Client]
C --> E[Placeholder Client]

StorageService.constructor()

ENV: SUPABASE_URL, SUPABASE_KEY SupabaseClient Instance

Micro-Logic:

  • Sử dụng helper isValidUrl (Regex/URL parser) để kiểm tra SUPABASE_URL.
  • Fail-safe: Nếu config lỗi, app không crash mà dùng placeholder.supabase.co.
  • Khởi tạo SupabaseClient bằng createClient.

upload(filePath, bucket, destination)

IN: string, string, string OUT: Promise<string> (path) 
graph LR
A[Check FS] --> B[Read Buffer]
B --> C[Supabase Upload]
C --> D[Return Data Path]

Micro-Logic:

  • Sử dụng fs.existsSync để tránh lỗi đọc file không tồn tại.
  • Dùng fs.readFileSync để chuyển file sang Buffer.
  • Cấu hình upsert: true để ghi đè file nếu đã tồn tại.
  • Mặc định contentType: 'audio/mpeg'.

uploadBuffer(buffer, bucket, destination, type?)

IN: Buffer, string, string, string OUT: Promise<string> 
graph LR
A[Receive Buffer] --> B[Supabase Upload]
B --> C[Return Path]

Micro-Logic: Bỏ qua bước đọc file từ ổ cứng, trực tiếp đẩy dữ liệu từ RAM lên Cloud. Hỗ trợ truyền contentType động.

getPublicUrl(bucket, path)

IN: string, string OUT: Promise<string> (URL) 
graph TD
A[Bucket/Path] --> B[Supabase getPublicUrl]
B --> C[URL String]

Logic: Gọi hàm generate URL đồng bộ từ SDK. Không cần chờ response từ network.

delete(bucket, path)

IN: string, string OUT: Promise<void> 
graph TD
A[Target Path] --> B[Supabase remove]

Impact: Xóa cứng tài nguyên trên Cloud Storage. Luôn bọc trong try/catch và log lỗi chi tiết.

Micro-Logic: Sử dụng Dependency Inversion (DI). Inject IStorageProvider thay vì class cụ thể, giúp tách biệt logic nghiệp vụ cleanup và implementation của Storage.
