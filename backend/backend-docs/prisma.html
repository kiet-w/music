Module: Prisma (Micro-Technical Detail)

Module prisma chịu trách nhiệm khởi tạo và quản lý kết nối cơ sở dữ liệu thông qua Prisma ORM, đảm bảo tính nhất quán và hiệu năng cho toàn bộ hệ thống.

1. Sơ đồ Hệ Thống (System Data Flow)

2. Chi tiết Module (PrismaModule)

File Path
src/prisma/prisma.module.ts
Scope
@Global()
Micro-Logic

Sử dụng decorator @Global() để biến PrismaService thành singleton toàn cục. Các module khác như SongsModule hay AlbumsModule có thể inject PrismaService mà không cần import lại PrismaModule trong mảng imports của chúng.

Impact

Giảm thiểu boilerplate code và đảm bảo chỉ có một instance duy nhất của PrismaClient quản lý connection pool, tối ưu hóa tài nguyên DB.

Micro-Logic

PrismaClient mặc định thực hiện lazy connection (kết nối khi có query đầu tiên). Việc ghi đè onModuleInit và gọi $connect() chủ động giúp ứng dụng thực hiện Fail-Fast: Nếu thông tin credentials sai hoặc Database server không khả dụng, NestJS sẽ báo lỗi ngay khi khởi động thay vì đợi đến khi người dùng thực hiện request đầu tiên.
