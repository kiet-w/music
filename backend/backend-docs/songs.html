Sơ đồ Hệ Thống (Module Flow)

POST createFromYoutube /youtube

Input (IN)
CreateSongYoutubeDto { url, title, artist, albumId }
Output (OUT)
SongResponseDto (HTTP 201)

Micro-Logic:

  • Nhận payload từ Client, thực hiện validation tự động qua CreateSongYoutubeDto.
  • Chuyển tiếp yêu cầu xử lý sang SongService.createFromYoutube.
  • Sử dụng plainToInstance để lọc dữ liệu trả về theo schema của SongResponseDto (đảm bảo tính bảo mật).
graph TD
A[Request Body] --> B{Validation}
B -- Valid --> C[Call SongService]
C --> D[Map to SongResponseDto]
D --> E[HTTP 201 Response]

Service: createFromYoutube

Service Call
songRepository.create, conversionQueue.add
Impact
Tạo track nháp & kích hoạt Worker tải nhạc

Logic Chi Tiết:

  • Album Check: Nếu albumId rỗng, tìm album "Default". Nếu không tồn tại, tạo mới với artist "Various Artists".
  • Draft Record: Tạo bản ghi Track với url: '' và sourceType: 'youtube'.
  • Job Queue: Đẩy job 'convert' vào BullMQ conversion queue để worker xử lý tải file ngầm.
graph LR
A[Start] --> B{Has albumId?}
B -- No --> C[Find/Create Default Album]
B -- Yes --> D[Use albumId]
C --> E[Create Track Record in DB]
D --> E
E --> F[Add Job to BullMQ]
F --> G[Return Song Object]

GET findAll /

In
None
Out
SongResponseDto[] (HTTP 200)

Logic:

  • Sử dụng CacheInterceptor để lưu cache kết quả, giảm tải DB.
  • Gọi SongService.findAll(), kết quả bao gồm thông tin album (Eager Loading).
graph TD
A[Request] --> B{Cache Hit?}
B -- Yes --> C[Return Cached Data]
B -- No --> D[Service.findAll]
D --> E[Store in Cache]
E --> F[Return Response]

Service: findAll / findOne

Logic:

  • Truy vấn DB qua Repository.
  • Sử dụng include: { album: true } để join bảng Album, lấy metadata đầy đủ cho bài hát.
graph LR
A[Call Repository] --> B[Prisma findMany/findUnique]
B --> C[Join Album Table]
C --> D[Return hydrated data]

DELETE remove /:id

Impact:

  • Xóa vĩnh viễn bài hát khỏi Database.
  • Nếu không tìm thấy ID, ném NotFoundException (HTTP 404).
  • Thành công trả về HTTP 204 No Content.
graph TD
A[Check Existence] --> B{Found?}
B -- No --> C[Throw 404]
B -- Yes --> D[Delete Record]
D --> E[HTTP 204]

PATCH moveToAlbum /:id/move

In (Body)
{ albumId: string }

Logic:

  • Cập nhật albumId mới cho track ID tương ứng.
  • Hữu ích khi người dùng muốn phân loại lại bài hát sau khi tải.
graph LR
A[Find Track] --> B{Exists?}
B -- Yes --> C[Update albumId]
C --> D[Return Updated Track]
B -- No --> E[Throw 404]
