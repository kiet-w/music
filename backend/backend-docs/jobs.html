ConversionProcessor Logic

process(job: Job) Hook
IN: { url: string, songId: string }
OUT: Promise<{ storagePath, publicUrl }>
Side Effects: Disk I/O, Supabase Upload, DB Update
graph TD
Start[Receive Job] --> Temp[Create Temp Dir]
Temp --> DL[Downloader.download]
DL --> UL[Storage.upload]
UL --> URL[Storage.getPublicUrl]
URL --> DB[Prisma.track.update]
DB --> Clean[Downloader.cleanup]
Clean --> End[Return Success]
DL -.->|Fail| Err[Cleanup & Throw]
UL -.->|Fail| Err
Micro-Logic:
  1. Prepare: Tạo thư mục /temp nếu chưa tồn tại.
  2. Step 1 (Downloader): Gọi yt-dlp để convert video URL thành MP3 cục bộ.
  3. Step 2 (Storage): Đẩy file MP3 vừa tạo lên bucket music với đường dẫn songs/${songId}.mp3.
  4. Step 3 (URL): Lấy URL công khai từ Supabase.
  5. Step 4 (Database): Cập nhật trường url trong bảng Track tương ứng với songId.
  6. Step 5 (Cleanup): Luôn xóa file temp dù thành công hay thất bại.
